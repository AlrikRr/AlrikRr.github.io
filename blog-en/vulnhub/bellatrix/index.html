üá¨üáß Vulnhub writeup : Bellatrix

Vulnhub challenge

February 22, 2021 AlrikRr

4 minute read

  • Twitter

  • Reddit

  • LinkedIn

LEVEL : Medium
TIME SPEND : 1h10
LINK : Bellatrix

NMAP

Okay let’s start with the beginning !

sudo nmap -A -sS bellatrix.vuln

As we can see, there are two main services :

PORT Status Banner
22 OPEN OpenSSH 8.3p1
80 OPEN Apache http 2.4.46

Web Application

The web application is using the http port 80. This page is a static html with a gif and a strange text with a .php at the end.

Unfortunatle, the page http://192.168.1.166/ikilledsiriusblack[...]ikilledsiriusblack.php is FORBIDEN.

After a few url manipulation, I figgured out that the index.php is actually ikilledsiriusblack.php.

Nothing more here, let’s take a look at the source code.

Well, there is something interesting, as we can see there is a PHP code, this code is showing us that we can display a file using the ?file= GET parameter.

We quickly realize that there is an LFI vulnerability, we can for example display the content of /etc/passwd.

Now that I have a good LFI exploit, I will try to find something to display because the goal of this exploit would be to have a shell on the machine.

I start a small fuzzing session with gobuster.

Fuzzing

gobuster -u http://bellatrix.vuln/ -w directory-list-2.3-medium.txt -x php,txt,html,db -fw

Okay well nothing interesting, considering the name of some pages I would need a special harry potter wordlist.

That’s ok, I’ll be able to test PHP wrappers and see if we can get the pages in base64.

PHP filter

php://filter/read=convert.base64-encode is working very well, I’m able to get the content of index.php and ikillsiriusblack.php but there is nothing more in those files.

I need to found how to put a php reverse shell or at least execute unix command.

Fuzzing LFI

I need a way to inject PHP code but at the moment I have no idea how to do it.

To continue, I’m fuzzing again but this time I’m looking to see if there are any system files that can be accessed by the www-data account. Maybe I’ll find my inspiration in that.

Indeed, I was right, we can see that the www-data user has access to the log file of the ssh service /var/log/auth.log.

http://bellatrix.vuln/ikilledsiriusblack.php?file=../../../..//var/log/auth.log

Then an idea comes to me. If I have the possibility to display the logs of the ssh service, I also have technically the possibility to inject anything I want. Let’s do a test. I will try to connect in SSH but my user will be a bit different:

 ssh '<?php system($_GET["c"]); ?>'@bellatrix.vuln

As you can see, my user is actually a PHP code that calls the system function with a GET variable. If everything goes as expected, then I will be able to inject the linux commands I want with this parameter because they will be executed by system().

As you can see, if I use the base64 filter to get the RAW data of the auth.log I can see my PHP code.

SSH log poisoning

Okay great, now let’s get started !

Let’s see if we missed folders or files with the command ls- al.

Indeed, as you can see, there is a folder named “secrets” in base64 and a gif.
Spoilers : the gif is just sirius black who dies.

On the other hand the secret folder contains two other files :

  • .secrets.dic : Well, well, well it’s a wordlist with a few words from the Harry Potter universe
  • Swordofgryffindor : It contains a username with the hash of his password.

SSH brute force

Well that’s not really a brute force because I have already guess the password of lestrange.
Obvious btw

System

Okay now I’ve got the user account, now we need to privesc and pwn that machine !

I hope that’s not gonna be so hard because the user have an Azkaban shell.

Since I have the lestrange’s password, I cant check the sudo rights and … Well …Se ha cometido un error !

The user can run vim with the root account without any password.

Privesc

I take a look at GTFOBins and search for a privsec on VIM.

This is what I got vim#sudo

Rooted !

Thanks

Thank you #BLY for this funny box !

I have learned a few things like SSH log poisoning, very interesting !
